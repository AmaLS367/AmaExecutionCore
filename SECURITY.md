## Supported Versions

| Version | Supported |
|---------|-----------|
| `main` branch | ✅ |
| `dev` branch | ⚠️ Development only |
| Older tags | ❌ |

## Reporting a Vulnerability

**Please do NOT open a public GitHub issue for security vulnerabilities.**

If you discover a security vulnerability, report it privately through one of the following channels:

| Channel | Contact |
|---------|---------|
| **Email** | amalsdev367@gmail.com |
| **Telegram** | [@Amanel0](https://t.me/Amanel0) |

### What to include

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact (e.g., unauthorized order placement, key exposure, etc.)
- Any suggested fix or mitigation

### Response timeline

- **Acknowledgement**: within 48 hours
- **Assessment**: within 5 business days
- **Fix & disclosure**: coordinated with the reporter

## Scope

The following are **in scope**:

- API authentication and authorization bypass
- Injection vulnerabilities (SQL, command)
- Sensitive credential exposure (API keys, secrets)
- Unintended trade execution or order manipulation
- Dependency vulnerabilities with exploitable impact

The following are **out of scope**:

- Bybit exchange platform vulnerabilities (report to Bybit directly)
- Issues in archived or unmaintained branches
- Theoretical vulnerabilities without a proof of concept

> This project handles real financial operations. Security is taken seriously. Thank you for responsible disclosure.
