# Руководство по API

## Состояние системы (Health)

### `GET /health`

Возвращает текущий режим работы и статус учетных данных (с частично скрытыми значениями).

## Выполнение сигналов

### `POST /signals/execute`

Основная точка входа для ядра исполнения.

Тело запроса:

```json
{
  "symbol": "BTCUSDT",
  "direction": "long",
  "entry": 100.0,
  "stop": 90.0,
  "target": 130.0,
  "reason": "optional",
  "strategy_version": "optional",
  "indicators_snapshot": {"optional": "json"}
}
```

Тело ответа:

```json
{
  "signal_id": "uuid",
  "trade_id": "uuid",
  "order_link_id": "string",
  "status": "order_submitted",
  "mode": "shadow",
  "replayed": false
}
```

Семантика идемпотентности:

- Публичный API выполняет дедупликацию по детерминированному отпечатку (fingerprint) нормализованного тела запроса.
- Правила нормализации:
  - `symbol` обрезается от пробелов и переводится в верхний регистр.
  - `direction` сопоставляется с точным значением перечисления (enum).
  - `entry`, `stop` и `target` преобразуются в отпечаток как нормализованные десятичные строки.
  - `reason` и `strategy_version` обрезаются от пробелов; пустые значения обрабатываются как `null`.
  - `indicators_snapshot` преобразуется в отпечаток как канонический JSON с отсортированными ключами.
- Если эквивалентный запрос повторяется в то время, когда связанная с ним сделка все еще находится в нетерминальном (незавершенном) состоянии, конечная точка возвращает `200` с теми же `signal_id` и `trade_id` плюс `replayed=true`.
- Если связанная сделка находится в статусе `order_pending_unknown`, повторный запрос инициирует еще одну проверку статуса на бирже и по-прежнему не отправляет второй ордер.
- Как только связанная сделка достигает терминального состояния, то же самое тело запроса может создать новые `Signal` и `Trade`.

Семантика ошибок:

- `422`: ошибка валидации рисков или превышение лимита воздействия (exposure).
- `423`: активна кнопка экстренной остановки (kill switch), период охлаждения (cooldown) или пауза автоматического выключателя (circuit-breaker).

## Безопасность

### `POST /safety/kill`

Активирует постоянную кнопку экстренной остановки и отменяет отложенные ордера на бирже в режимах `demo` или `real`.

### `POST /safety/reset`

Сбрасывает состояние постоянной остановки/паузы после проверки оператором.

### `GET /safety/status`

Возвращает:

- `kill_switch`
- `pause_reason`
- `cooldown_until`
- `manual_reset_required`
- `trading_mode`
