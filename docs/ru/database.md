# Руководство по базе данных

## Основные таблицы

### `signals`

Хранит нормализованные намерения стратегии:

- symbol
- abstract direction
- reason
- strategy version
- indicator snapshot

### `trades`

Хранит полный жизненный цикл исполнения:

- снимок риска на момент входа
- ID ссылки на ордер входа / ID ордера на бирже
- ID ссылки на ордер закрытия / ID ордера на бирже
- цены исполнения, комиссии, проскальзывание (slippage)
- текущий статус жизненного цикла
- реализованный PnL и время удержания (hold time)

Важные дополнения статусов:

- `ORDER_PENDING_UNKNOWN`
- `POSITION_CLOSE_PENDING`
- `POSITION_CLOSE_FAILED`
- `PNL_RECORDED`

### `trade_events`

Журнал аудита (только для добавления) для изменений жизненного цикла сделок.

Каждая строка записывает:

- владеющий `trade_id`
- `event_type` (тип события)
- `from_status`
- `to_status`
- необязательные метаданные, описывающие источник перехода

Среда выполнения в настоящее время записывает:

- `trade_created` когда новая строка `Trade` впервые сохраняется
- `status_transition` всякий раз, когда существующая среда выполнения перемещает сделку между состояниями жизненного цикла

`trades.status` остается изменяемым снимком текущего состояния. `trade_events` является источником истории переходов для аудита и реконструкции.

### `daily_stats`

Агрегирует реализованные ежедневные результаты, используемые автоматическим выключателем (circuit breaker):

- количество сделок
- последовательные убытки (consecutive losses)
- процент ежедневных убытков (daily loss percentage)
- флаг автоматического выключателя

Аналитические поля, поддерживаемые средой выполнения:

- `gross_pnl`
- `total_fees`
- `net_pnl`

Аналитические поля, отложенные средой выполнения:

- `starting_equity`
- `ending_equity`

Эти отложенные поля остаются в схеме, но в настоящее время не заполняются кодом времени выполнения.

Отложенная аналитика на уровне сделки:

- `mae` (Maximum Adverse Excursion)
- `mfe` (Maximum Favorable Excursion)

### `system_events`

Журнал аудита для кнопки экстренной остановки (kill switch), автоматического выключателя, периода охлаждения (cooldown) и операционных ошибок.

### `safety_state`

Таблица в стиле синглтона для глобального состояния безопасности выполнения. Она сохраняет условия остановки бота при перезапусках процесса.

## Поддерживаемая матрица тестов

Репозиторий намеренно поддерживает два уровня тестов БД с разными целями.

### Быстрые тесты SQLite

Тесты по умолчанию используют `sqlite+aiosqlite` "в памяти" из `tests/conftest.py` и создают таблицы через `Base.metadata.create_all`.

Что они валидируют:

- поведение сервисов и маршрутизаторов
- ORM маппинги, используемые текущей средой выполнения
- быструю обратную связь для логики выполнения, безопасности, сверки и бэктестирования

Что они не валидируют:

- корректность миграций Alembic
- типы столбцов, ограничения и поведение DDL, специфичные для PostgreSQL
- расхождение между моделями во время выполнения и перенесенной производственной схемой

### Интеграционные тесты PostgreSQL + Alembic

`tests/postgresql/` — это опциональный путь интеграции для реального стека персистентности (persistence). Для него требуется `TEST_POSTGRESQL_URL`, и он запускает `alembic upgrade head` перед выполнением тестов.

Что они валидируют:

- создание схемы через миграции, а не через `Base.metadata.create_all`
- совместимость текущего кода ORM/среды выполнения с перенесенной схемой PostgreSQL
- критические потоки сохранения и API против таблиц и ограничений PostgreSQL

Текущее покрытие включает:

- сохранение торгового журнала для `Signal`, `Trade`, `SafetyState`, `DailyStat` и `trade_events`
- поведение воспроизведения `POST /signals/execute` в перенесенной схеме
- конечные точки безопасности против сохраненного состояния PostgreSQL
- базовый жизненный цикл закрытия теневой сделки против перенесенной схемы

Ограничения:

- эти тесты не являются полной сквозной производственной симуляцией (end-to-end production simulation)
- они не доказывают поведение на стороне биржи, работу сети или корректность сверки с Bybit
- целевая база данных должна быть одноразовой; фикстура "opt-in" сбрасывает схему `public` перед миграциями
